# AGENTS.md

本仓库全程由 AI Agent 独立开发与维护。研发核心原则不以「快速迭代完工」为目标，而是保障每一次代码变更**逻辑清晰、可追溯验证、长期可维护**。

## 工作原则

- 优先修复问题根本成因，杜绝仅屏蔽告警、治标不治本的临时规避方案。
- 开发过程中遇显性错误、异常缺陷，应同步联动修复，而非临时绕过搁置。
- 每项开发任务完成后，必须执行对应模块测试，杜绝新增隐性问题与功能退化。
- 若修改脚本、路由规则、环境变量、默认端口、接口契约等核心配置，需同步联动更新 `README`、`.env.example`、OpenAPI 文档、配套说明及相关测试用例。
- 控制单文件代码体量：常规文件行数严控在 400 行以内，上限不超过 500 行，便于 Agent 阅读与长期维护；特殊场景需超限时，须有合理的理由。
